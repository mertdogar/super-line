# super-line

Strictly-typed realtime data bus for TypeScript — one contract for every pattern on the wire (requests · events · subscriptions), end-to-end types, no codegen. Same API on one node or a cluster. pnpm monorepo.

## Commands (run from repo ROOT)

```bash
pnpm test         # vitest run — the REAL suite (~399 tests). `pnpm -r test` runs NOTHING.
pnpm typecheck    # root tsc — the REAL check. `pnpm -r typecheck` only covers 2 of 18 projects.
pnpm lint         # oxlint (NOT eslint)
pnpm build        # pnpm -r build (tsup)
pnpm docs:dev     # VitePress dev server
pnpm docs:build
```

## Layout

`packages/{core,server,client,react,control-center}` · `packages/transport-{websocket,http,libp2p,loopback}` (pluggable client↔server transports) · `packages/adapter-{redis,libp2p,rabbitmq,zeromq}` (server↔server fan-out) · `packages/store-{memory,sync,sqlite,pglite,sync-pglite}` (legacy `ServerStore` backends, **being retired** — ADR-0007; `store-sync-libsql` already deleted) · `packages/collections-{memory,sqlite,pglite}` (typed row collections — `CollectionStore` backends) · `packages/collections-crdt-{memory,libsql}` (CRDT document collections — `CrdtCollectionStore` backends; `collections-crdt-pglite` not yet built) · `packages/tanstack-db` (TanStack DB sync adapter) · `packages/plugin-{inspector,auth}` (paired plugins: Control Center inspector · first-party authentication) · `examples/` · `docs/` (VitePress) · `skills/super-line/` (published agent guide for *users* of super-line, not dev-context for this repo).

- `defineContract` lives in `@super-line/core`; `createSuperLineServer` in `/server`, `createSuperLineClient` in `/client`, `createSuperLineHooks` in `/react`. The wire error type is `SuperLineError`.
- **Client↔server transport is pluggable.** Server takes `transports: [...]`, client takes `transport:`; `authenticate` receives a normalized `Handshake`. `webSocketServerTransport`/`webSocketClientTransport` (default) live in `@super-line/transport-websocket`; HTTP-SSE/long-poll in `/transport-http`; libp2p in `/transport-libp2p` (bring-your-own node); in-memory `/transport-loopback` for tests. Transport interfaces (`RawConn`/`ServerTransport`/`ClientTransport`/`Handshake`) are in `@super-line/core`. Migration notes: `TRANSPORT-MIGRATION-NOTES.md` at repo root.
- `docs/reference/` is typedoc-GENERATED — never hand-edit; it regenerates from source on `docs:build`.

## Gotchas

- **Tests and typecheck only work from root**, not via `pnpm -r` (no package has a `test` script; only 2 have `typecheck`). `pnpm -r test` silently passes while running nothing.
- `packages/server/test/reconnect.integration.test.ts` → "auto-reconnects after an abrupt drop" is timing-flaky under full parallel load; it passes in isolation. Don't chase it as a regression.
- ESM-only packages. Lint/format is oxlint/oxfmt, not ESLint/Prettier.

## Architecture

One contract is the single source of truth (`defineContract`), imported by both server and client, split by direction (`clientToServer`/`serverToClient`) and scoped by role (a `shared` base + per-role blocks). **Server-authoritative**: the server owns rooms/topics, authorizes subscribes, and validates every inbound message; roles are fixed at connect by `authenticate(handshake)` (cross-role calls → `NOT_FOUND`). Three wire patterns: requests (req/res), events (server push), topics (client subscribe). The **client↔server wire is a pluggable transport** (WS/HTTP/libp2p/loopback — see Layout). Separately, **server↔server cross-node fan-out** goes through a pluggable `Adapter` (in-memory default; Redis/libp2p/rabbitmq/zeromq adapters ship) plus a server-side cluster bus (`server.publish`/`server.subscribe` with local echo). Don't confuse the two: transports carry client↔server bytes; adapters carry node↔node fan-out (but see Stores — `clustering: 'self'` stores bypass the adapter entirely).

**Stores subsystem.** A fourth wire pattern: synced server-authoritative state. The server holds a `ServerStore` per namespace; `srv.store(ns).open(id)` returns a `ServerReplica` the server can co-write, and clients/react open the same resource as a handle (`store(ns).open(id)` → `ResourceHandle` with `data`/`set`/`update`/`delete`; `useResource()` in react). Stores vary on three axes: **model** (`lww` plain LWW vs `crdt` Yjs/super-store), **durability** (in-memory / better-sqlite3 / libsql-Turso / Postgres), and **clustering** — `'relay'` fans changes out over the server↔server Adapter (so a relay store still needs an adapter for >1 node), while `'self'` stores own a central backend + a per-node replica (Postgres + Electric→PGlite) and need **no adapter**. The 6 packages: `store-memory` (LWW·memory·relay, the default; pairs with `memoryStoreClient`), `store-sync` (CRDT·memory·relay; `DocOptions {mode, opaque}`; the engine libsql/pglite reuse), `store-sqlite` (LWW·durable·relay), `store-sync-libsql` (CRDT·durable·relay; **async factory** `await libsqlSyncStore(...)`; snapshot-per-resource, history-preserving rehydrate), `store-pglite` (LWW·**self**), `store-sync-pglite` (CRDT·**self**). **Deletion fan-out:** `srv.store(ns).delete(id)` publishes a wire `SDeleteFrame` (`'sdel'`) cluster-wide; observe via `ServerStore.onDelete`, and on the client/react via `ResourceHandle.deleted` / `useResource().deleted` (a deleted resource otherwise reads as a silent empty snapshot). Design docs at repo root: `PLAN-store-sync-libsql.md`, `PLAN-store-pglite.md`, `PLAN-store-sync-pglite.md`.

**Collections subsystem (ADR-0006).** The relational successor to the LWW store family: typed rows declared **on the contract** (`collections: { name: { schema, key, references? } }` in `defineContract`), so the server validates every write and types flow end-to-end (`RowOf<C,N>`). super-line is the server-authoritative **sync source**; **TanStack DB is the client query engine** (joins/live-queries/optimism) via the `@super-line/tanstack-db` adapter — there is no native query engine. Core owns a query **IR** + one evaluator (`packages/core/src/query.ts`: `evalExpr`/`applyQuery`/`matchesFilter` + `and`/`or`/`not`/`eq`/`in`/`like`… builders and `and`/`orFilters`) shared by change-routing, scan snapshots, and client re-filtering. Wire frames: `csub`/`cuns`/`cbat`/`cchg`. One `CollectionStore` backend per server serves all collections (single tx domain → atomic cross-collection batches); `collections-memory`/`-sqlite` are `relay`, `collections-pglite` is `self` (central Postgres + Electric→PGlite, same topology as store-pglite; `live.changes` carries only changed columns + the key, so a synthetic `pk` encodes `collection`+`id`). **Row security** = server-side `policies: { ns: { read(principal,ctx)→IR filter | undefined, write(principal,op,next,prev)→bool } }`, deny-by-default (omit `read`/`write` ⇒ denied). Routing is stateless per-connection beyond the predicate (deliver if pre- OR post-op row matches → client re-filters; prev-less `self` deletes broadcast to all subs). **Advisory FKs**: `references` metadata + opt-in `checkReferences` server flag (no cascades). `srv.collection(n)` co-writes (policy-free, schema-validated); `client.collection(n).subscribe(query)` → non-optimistic live row-set + `insert`/`update`/`delete`/`batch`; react `useCollection`; inspector `listCollections`/`queryCollection` + Control Center Collections view. Client contract: **await `sub.ready` before depending on live delivery** (frames process concurrently). Design + decisions: `PLAN-collections.md`, `docs/adr/0006-collections-are-on-contract-typed-rows.md`; guide `docs/guide/collections.md`; example `examples/collections`.

**CRDT document collections (ADR-0007).** The CRDT doc family folds INTO collections — one `collection(n)` concept, two consistency models (LWW rows · CRDT docs). A CRDT collection is declared on the contract by a `crdt` key (`collections: { scenes: { schema, crdt: DocOptions } }`, no `key` — id is external); it is **opened by id, not queried** (whole-doc merge). The headline reversal of ADR-0003/0006: opaque CRDT deltas ARE now validated — **validate-before-commit** at the ingress node (the backend merges the delta onto a scratch copy, snapshots to plaintext, the server validates against the contract schema, then commits + fans out only if valid; relay nodes trust already-validated deltas). Reject → the writer resyncs (server-push resync deferred — see PLAN). Access = guard-shaped `CrdtCollectionPolicy` (`read(principal,id,snapshot?)→bool`, `write(principal,id)→bool`, deny-by-default), NOT the RLS filter shape. **Creation is server-authoritative** (`srv.collection(n).create(id,data)`; clients open existing docs; nonexistent → `NOT_FOUND`). Backend seam: `CrdtCollectionStore` (keyed by `(collection,id)`, no ACL, validate hook) — a SEPARATE backend from `CollectionStore` (`crdtCollections:` server option; CRDT never joins a cross-collection atomic batch). Client: `client.collection(n).open(id)` → `DocHandle` (relocated `ResourceReplica` surface); needs `crdtCollections: crdtCollectionsClient()` (the universal client engine); react `useDoc`. Wire frames: `cdopen`/`cdwr`/`cdchg`/`cddel`/`cdclose` (mirror the store frames), per-doc channel `d:<n>:<id>`. Packages: `collections-crdt-memory` (relay, + the universal `crdtCollectionsClient`), `collections-crdt-libsql` (durable·relay, `await crdtLibsqlCollections`, snapshot-per-doc), `collections-crdt-pglite` (**self** — NOT YET BUILT). Inspector: CRDT collections surface in `listCollections` (synthetic `id` key) + `queryCollection` synthesizes `{id, ...snapshot}` doc-rows, so the Control Center Collections view browses them with no CC change. **Store family is being retired** (`store-sync-libsql` already deleted); the `store(n)` API + `store-{memory,sqlite,pglite}` + `store-sync`/`-pglite` retire once the pglite tier is relocated. Design: `PLAN-collections-crdt.md`, `docs/adr/0007-crdt-docs-are-typed-collections.md`.

**Contract-fragment plugins + auth.** ADR-0005's runtime-only plugin boundary grew a **contract-time half**: `defineContract({ plugins: [contractPlugin] })` merges a plugin's collections/roles/shared surface INTO the contract via a plain **intersection** (`ResolveContract`), so `RowOf`/`client.collection`/per-role `Requests` infer from the single materialized contract with zero type-threading — existing callers untouched (`defineContract` is overloaded; no-plugins = identity). Server-side, a plugin can also contribute `policies` (merged), and `SubtractHandlers` now drops **fully-plugin-owned** blocks to optional (so a host needn't pass empty `shared:{}`/`guest:{}`). Built on this: **`@super-line/plugin-auth`** (subpath package: `.` contract · `/server` · `/client` · `/react`) — first-party auth = email/password (scrypt), stateful server-issued **sessions** (256-bit token, sha256-at-rest), **data-driven roles** (`roles[]` on the user row; only `guest` hardcoded), **API keys** (`slp_`, fixed role), **JWT** (jose HS256, on-demand `getToken` + stateless `?jwt=` connect), `authKit.revoke(userId)` (session-delete + cluster-wide `toUser().disconnect()`), and password reset via a host callback. All identity lives in collections (`users` public; `credentials`/`sessions`/`apiKeys`/`passwordResets` deny-all). Wiring is 3 touch-points: `plugins:[authContract()]` on the contract, `authenticate: authKit.authenticate` + `identify` + `plugins:[authKit.plugin]` on the server, `createAuth()`/`authClient()` on the client (hides super-line's guest→user reconnect since role is frozen at connect). Design: `PLAN-plugin-auth.md`; guide `docs/guide/plugin-auth.md`; examples `examples/auth` (CLI) + `examples/collections-chat` (real login).
